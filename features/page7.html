Functional Programming

Functional programming is a programming paradigm.

Functional programming principles:

  • Avoid side-effects (changing application state)
  • Avoid mutating data
  • Use pure functions

Features that support Functional Programming: Pure Functions, First-Class Functions, Higher-Order Functions.

Pure Function definition:

  • It returns the same result if given the same arguments
  • It does not cause any side effects (it does not change global variables or the application state)
  • It does not change any of the parameters that get passed to it by reference (object and arrays)

Benefits of using the Pure Functions: code is easier to test, code is more predictable.

First-Class Functions - functions that can be treated as values (means they can be assign to variables and object properties, passed to functions or returned from functions).

Higher-Order Function - a function that takes another function as a parameter or returns a function.
