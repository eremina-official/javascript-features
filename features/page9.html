Composition and Inheritance

Fundamentally inheritance is a code reuse mechanism, a way of different objects to share and reuse code.

Difference between Class and Prototypal inheritance:

In the programming languages that support classical inheritance Class is like a blueprint (template) for object creation, a description of the object to create. Classes can inherit from classes creating bound class taxonomies. A new object is usually instantiated via constructor functions with the 'new' keyword.

In JavaScript a prototype is an object instance, so objects inherit directly from other objects. Objects are usually instantiated via factory functions, object literals, Object.create() and Object.assign() methods.

Object.create() - creates a new object using the object passed as an agrument as a prototype.

Object.assign() - copies properties of one or more source objects to target object and returns target object.

Prototype in JavaScript is: each object has a property called prototype which points to another object from which the initial object can inherit properties and methods. This another object is also called prototype.

In JavaScript prototypal inhertitance can work in few ways:

  • Properties and methods can be copied from one object to another (usually via Object.assign()). This is different from a prototype delegation in a way that it increases the memory usage, while the properties are copied for every new instance, but not delegated as with prototype delegation.
  • Prototype delegation - properties and methods can be delegated to prototype. If a method is not found on the object, JavaScript engine looks up for this method on the object's protopype and then further on the prototype chain.
  • Functional inheritance - via Factory Functions. Factory function is set up to return an object and does not use the 'new' keyword.

With composition we first define features (properties and methods) and then can compose objects with these features.
