Strict mode

What strict mode does:

  • changes some silent JavaScript errors into throw errors: f.e. in the strict mode it is not possible to accidentaly create a new global variable by mistyping in an assignment, create properties with same keys on one object, assign to non-writable property or delete undelitable property.
  • fixes mistakes that make it difficult for JS engines to perform optimisations.
  • prohibits some syntax which is likely to be used in future versions of ECMAscript (f.e. adds reserved words like let etc).

'use strict' statement applies for entire scripts or for individual functions, it does not apply for block statements. For the 'use strict' statements to work for the entire stript it should be placed at the very top of the js file.
