In concurrent systems computations can be executed concurrently (during overlapping time periods) instead of sequentially (one completing before the next starts).

Concurrency model specifies how the different tasks inside the system are coordinated.

JavaScript code runs single-threaded.

JavaScript is synchronous by default.

JavaScript runtime (browser or Node.js) provides means for JavaScript to run asynchronously.

Inside the browser JavaScript Engine runs JavaScript code synchronously and single-threaded. It keeps Heap (area where objects are stored) and Call Stack (keeps a track of all execution contexts).

Browser provides a set of APIs (DOM, ajax, setTimeout) that allow for asynchronous code running. These APIs provide threads which are separate from that thread where the main JS code runs and run all asynchronous tasks.

Tasks that run asynchronously: user-defined events, DOM events, ajax requests, setTimeout and setInterval methods, I/O (reading from file system etc).

All function calls that should be processed asynchronously are placed it the Message Queue. When the Call Stack becomes empty, JS runtime moves the oldest function from the Queue to the Call Stack.

Event Loop is a mechanism which constantly checks if there are tasks in the Message Queue and pushes the oldest task the Call Stack as soon as the Stack is empty.

Callback is a function that is passed as a parameter to a Higher-Order Function. Callbacks in JavaScript can run synchronously (when calling the argument function is performed as a part of normal execution of statements) or asynchronously (when the callback function performs an asynchronous operation).
