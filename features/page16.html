Object-oriented Programming Principles

Principles of Object-oriented Programming:

  • Abstraction - is a process of presenting only relevant features of an object and hiding the features which are less important. F.e. if an object represents a real world item, this object describes only certain features of this item and ignores features that are currently not important. Abstraction helps to reduce complexity.
  • Encapsulation - is an idea of hiding object's properties and methods inside the object so that they could not be modified from outside.
  • Inheritance - is an idea that objects can inherit properties and methods from other objects.
  • Polymorphism - is an idea that methods can have the same name but different implementations.
